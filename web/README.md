# Универсальный web/PWA-прототип

Статическая адаптивная версия «Было Тут» для браузеров Android, iPhone и ПК.

## Локальный запуск

Дважды щёлкните `run-web.bat` или выполните:

```powershell
cd web
python -m http.server 8080
```

Если Python отсутствует, но установлен Node.js:

```powershell
cd web
node local-server.mjs
```

`run-web.bat` сам выбирает доступный вариант; устанавливать npm-пакеты не требуется.
Сервер запускается скрыто, поэтому чёрное окно закрывается сразу. После работы при желании
запустите `stop-web.bat`, чтобы остановить скрытый локальный сервер.

Откройте `http://localhost:8080`. Для доступа с другого устройства разместите папку
на HTTPS-хостинге: камера и геопозиция не работают на обычном незащищённом HTTP.

## Реализовано в прототипе

- выбор JPEG/HEIC и локальная попытка чтения EXIF;
- текущая позиция, ручные координаты и выбор точки на карте;
- видоискатель и снимок в памяти страницы;
- прямой поиск ближайших фотографий через PastVu API;
- список, карта, направление из PastVu и показ двух фото рядом;
- PWA-манифест и локальный кэш файлов приложения.

Современное фото не отправляется. В PastVu передаются координаты и параметры поиска.
Карта загружает нейтральные тайлы CARTO на данных OpenStreetMap для видимой области и
автоматически пробует стандартный OSM-источник как резерв. Если обе подложки недоступны,
маркеры, выбор точки и масштаб продолжают работать на нейтральном фоне. Фиксированные версии Leaflet 1.9.4
и exifr 7.1.3 находятся в `vendor/`; их MIT-лицензии сохранены рядом.

Используется открытый API PastVu. Web-версия не является официальным приложением PastVu.
Права на исторические изображения принадлежат их соответствующим правообладателям; название
результата ведёт на страницу источника, а каждая карточка имеет подпись `Источник: PastVu`.

## Ограничения прототипа

- браузер не сканирует всю галерею: файл выбирает пользователь;
- автоматическое длительное фоновое обновление позиции не выполняется;
- для полноценной установки на iPhone желательно добавить PNG-иконки 180/192/512;
- поведение HEIC/EXIF зависит от браузера и требует проверки на реальных iPhone и Android.
